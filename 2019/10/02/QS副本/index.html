C# ref参数问题
在C#中，你给一个方法传输值类型参数时，实际上是使用的这个参数的一个副本，就是将原来的变量复制一份，然后传给一个方法，让其进行操作。所以在方法内部对参数的修改等不会对原来的参数造成影响（这个其实就是值类型（传值）和引用类型（指针/传址）的区别）。但是有些时候，又需要这种影响。ref的作用就是这个。它将变量本身而不是副本传给方法，所以对参数的修改原来变量的值。

此外，out修饰符也起到这个作用，二者的区别就是ref需要初始化，而out修饰的参数不需要初始化。

为了达到 name 唯一，1.name unique 2.加编号(pk),并又含有实际意义 ，对于重名数据定位友好。
如果用id （id int primary key identity(1,1)）代替 编号 ， 不能唯一标识name ，会有重名，对于重名不好定位。

对于那些如果设置编号有较大意义（如可以用作连接查询，那就优先这种），此外，就采用id吧。

总之，对于有唯一性的数据，采用 标识号+数据名，当然后端有后端的方法，前端有前端的方法。

伪元素 和 伪类 区别

伪类选择元素基于的是当前元素处于的状态，或者说元素当前所具有的特性，而不是元素的id、class、属性等静态的标志。由于状态是动态变化的，所以一个元素达到一个特定状态时，它可能得到一个伪类的样式；当状态改变时，它又会失去这个样式。由此可以看出，它的功能和class有些类似，但它是基于文档之外的抽象，所以叫伪类。
与伪类针对特殊状态的元素不同的是，伪元素是对元素中的特定内容进行操作，它所操作的层次比伪类更深了一层，也因此它的动态性比伪类要低得多。实际上，设计伪元素的目的就是去选取诸如元素内容第一个字（母）、第一行，选取某些内容前面或后面这种普通的选择器无法完成的工作。它控制的内容实际上和元素是相同的，但是它本身只是基于元素的抽象，并不存在于文档中，所以叫伪元素。
